## 1. 产品功能架构概述

### 1.1 产品解决的业务问题

在企业制造过程质量审核场景中，审核人员需要依据分散的审核标准，对业务系统数据及线下文档材料进行逐条比对分析，识别不符合项并输出审核结论。当前面临的核心业务问题包括：

- 审核规则分散，执行依赖个人经验，标准不统一
- 人工整理材料、比对标准、编写报告效率低
- 复杂场景下问题识别存在遗漏风险
- 审核经验难以系统化沉淀与复用
- 集团层面缺乏对多工厂审核进展与风险的整体掌握

本产品以 **企业级 AI 智能工作平台** 定位，将 AI 能力嵌入审核全链路，辅助专业人员完成复杂审核任务，而非替代其专业判断。

### 1.2 核心业务链路

```
创建审核项目 → 配置区域/项目/子要素及一对一数据源 → 启动 AI 审核
→ 解析/匹配/问题识别与描述及评分 → 人工复审（改描述与打分）
→ 报告中心四类产出（可独立导出）
```

| 阶段 | 业务能力 | 核心产出 |
|------|----------|----------|
| 创建与配置 | 审核项目管理 | 审核项目（名称自动生成）；区域→项目→子要素树；子要素级数据源 |
| AI 审核执行 | AI 审核执行能力 | 解析结果、匹配标准、问题描述与评分建议 |
| 人工复审 | 审核结果管理 | 经修改确认的问题描述与打分 |
| 报告输出 | 报告中心 | ① BIP问题管理表 ② 打分结果 ③ 区域/工厂/集团共性问题总结 ④ 综合性报告 |

### 1.3 AI 在系统中的作用

| AI 介入点 | 说明 | 人工 |
|-----------|------|------|
| 执行阶段：资料解析 | 解析子要素对应资料/数据 | 配置数据源、现场取证 |
| 执行阶段：标准匹配 | 匹配知识库审核标准 | 确认审核对象树 |
| 执行阶段：问题识别与描述及评分 | 生成问题描述与打分建议 | 人工复审中修改描述与打分 |
| 报告中心：BIP 问题管理表 | 形成问题清单类产出 | 可改、可独立导出 |
| 报告中心：打分结果 | 形成打分交付物 | 可改、可独立导出 |
| 报告中心：共性问题总结 | 区域/工厂/集团级汇总 | 可触发生成、可改、可导出 |
| 报告中心：综合性报告 | 汇总性报告 | 可触发生成、可改、可导出 |

**核心原则：AI 提供建议，人工最终确认。**

## 2. 用户角色与功能关系

两类角色使用 **同一套功能与页面**，差异仅在 **数据权限范围**（管理人员通常可见更广工厂/项目数据）。

### 2.1 共同使用的功能

| 使用功能 | 使用目标 |
|----------|----------|
| 审核项目列表 | 创建任务；查看统计（含状态分布、工厂情况、问题趋势）；按状态进入 |
| 审核项目工作台 | 配置子要素与数据源；启动 AI；查看 AI 三步过程 |
| 人工复审 | 修改问题描述与打分结果 |
| 报告中心 | 四类产出查看、修改、独立下载 |
| 移动端现场取证 | 人工录入类子要素的线下表单拍照上传 |

## 3. 一级功能架构

一级功能按用户任务驱动，共 **五大能力域**（已取消独立驾驶舱）。

---

### 3.1 审核项目列表（含态势统计）

**能力定位：** 审核任务入口，并承接原驾驶舱中的高价值统计，便于在同一界面掌握负荷与进展。

| 功能 | 能力说明 |
|------|----------|
| 创建审核项目 | 必须提供创建入口；项目名称等标识自动生成；无独立「基础信息」配置步 |
| 项目列表与四态 | 配置中 / AI审核中 / 人工复审中 / 报告已生成 |
| 数据卡片 | 审核项目总数、AI审核中、人工复审中、已完成审核 |
| 图表统计 | 审核状态分布、工厂审核情况、问题趋势等 |

---

### 3.2 审核项目管理（工作台配置与启动）

**能力定位：** 承载从创建后配置到启动 AI 的组织工作；配置以「区域→项目→子要素」树为核心。

**解决的业务问题：** 审核对象层级清晰，数据源必须与子要素一一对应，系统无法在项目级批量上传后自行匹配。

#### 3.2.1 审核项目创建

| 能力 | 说明 |
|------|------|
| 创建入口 | 列表页必须提供创建；创建后进入工作台配置 |
| 自动生成标识 | 项目名称等自动生成；**无独立「基础信息」配置步骤** |
| 周期 | 以日期选择方式确定（月 / 季 / 半年 / 年等粒度，交互细则在实现阶段定） |
| 基地等 | 基地下拉等结构化选择（与树节点字段配合） |

#### 3.2.2 审核范围与子要素配置（左树右配）

| 能力 | 说明 |
|------|------|
| 左侧对象树 | **区域 → 项目 → 子要素**（示例：零部件 → 质量数据采集 → 质量监视测量设备配置） |
| 右侧子要素配置 | 展示该子要素匹配的知识库审核标准（如设备管理审核规则 V1.0） |
| 节点状态 | 已完成配置 / 缺少数据源 / 未选择 |

#### 3.2.3 数据源配置（与子要素一对一）

每个子要素单独配置数据源，支持：

| 数据来源 | 说明 |
|----------|------|
| 文件资料 | 通用上传形态；上传与该子要素绑定 |
| 系统数据 | 配置取数来源，并展示具体字段含义（如问题记录时间） |
| 人工录入 | 无法走文件/系统自动取数、依赖线下表单的子要素；现场拍照等取证通过 **移动端** 完成 |

#### 3.2.4 审核启动

| 能力 | 说明 |
|------|------|
| 完备性检查 | 校验子要素选择与数据源是否完备 |
| 启动确认 | 弹窗提示：启动后范围与数据源不可修改 |
| 锁定 | 启动后锁定配置，进入 AI 审核 |

---

### 3.3 AI 审核执行能力

**能力定位：** 启动后自动执行；本阶段只完成三步，并展示过程详情（Demo 可用静态/分步呈现，不强调炫技动效）。

```
资料解析 → 标准匹配 → 问题识别与描述及评分
```

| 步骤 | 能力说明 |
|------|----------|
| 资料解析 | 解析该子要素绑定的文件/系统/人工录入资料 |
| 标准匹配 | 命中并展示具体审核标准 |
| 问题识别与描述及评分 | 生成问题描述与打分建议，供人工复审修改 |

**结束后直接进入人工复审，不设「分析完成」中转页。**

> 区域/工厂/集团共性问题总结、综合性报告等在 **报告中心** 触发，不在本能力域自动跑完。

---

### 3.4 人工复审（原审核结果管理）

**能力定位：** 对人机协同产出的问题描述与打分结果进行审阅与修改。

| 能力 | 说明 |
|------|------|
| 问题与依据查看 | 查看描述、所属子要素、来源、标准与分析线索 |
| 修改问题描述 | 调整 AI 生成表述 |
| 修改打分结果 | 调整评分 |
| 进入报告中心 | 复审后进入四类产出 |

---

### 3.5 报告中心

**能力定位：** 四类 AI 产出并列管理；均可独立下载；均支持人工修改（模板由业务方提供）。

| 产出 | 说明 |
|------|------|
| ① 问题描述 / BIP 问题管理表 | 问题清单类交付 |
| ② 问题打分结果 | 打分类交付 |
| ③ 共性问题总结 | 区域级 / 工厂级 / 集团级汇总 |
| ④ 综合性报告 | 汇总性审核报告 |

---

### 3.6 移动端现场取证

**能力定位：** 服务「人工录入」类子要素的线下表单现场取证（拍照上传等），汇入 Web 复审链路。

| 功能 | 能力说明 |
|------|----------|
| 我的审核任务 | 查看相关任务 |
| 现场拍照/上传 | 针对人工录入类子要素上传线下表单影像等 |

## 4. AI 能力架构

### 4.1 知识库能力

子要素配置时展示并匹配审核标准；执行阶段按对象检索标准条款。

### 4.2 大模型能力（介入点）

| 介入点 | 所在环节 |
|--------|----------|
| 解析 / 匹配 / 问题识别与描述及评分 | AI 审核执行 |
| BIP 问题管理表、打分结果、共性总结、综合报告 | 报告中心（可单独触发与导出） |

### 4.3 人机协同

| 环节 | AI | 人工 |
|------|----|------|
| 配置 | 展示匹配标准 | 选树、配数据源、启动确认 |
| 执行 | 三步分析 | 查看过程详情 |
| 复审 | 提供描述与打分建议 | 修改描述与打分 |
| 报告 | 生成四类产出 | 修改并独立导出 |

## 5. 功能模块关系

### 5.1 完整业务链路

```
列表创建审核项目（名称自动生成）
    ▼
工作台配置（左：区域→项目→子要素；右：标准 + 一对一数据源）
    ▼
启动确认并锁定
    ▼
AI 审核：资料解析 → 标准匹配 → 问题识别与描述及评分
    ▼
人工复审：修改问题描述与打分（无中转页）
    ▼
报告中心：四类产出查看 / 修改 / 独立导出
```

### 5.2 角色与模块

两类角色参与同一链路；管理人员因数据权限可见范围更大。无独立驾驶舱模块。

### 5.3 模块依赖

```
审核项目列表（含统计）──创建/进入──► 审核项目工作台
审核项目工作台 ──启动──► AI 审核执行 ──直达──► 人工复审 ──► 报告中心
移动端现场取证 ──汇入──► 人工复审（人工录入类证据）
```
