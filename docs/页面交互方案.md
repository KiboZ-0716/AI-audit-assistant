# 1. 交互设计目标

向客户展示：

1. 从 **创建审核项目** 到 **报告下载** 的完整闭环  
2. **子要素级** 标准匹配与数据源配置（一对一）  
3. AI 自动完成：解析 → 匹配 → 问题识别与描述及评分  
4. **人工复审** 可改问题描述与打分  
5. 报告中心四类产出可独立导出、可再修改  

不展示独立驾驶舱；管理与执行人员界面一致，仅数据范围不同（Demo 可用说明或切换模拟）。

---

# 2. 核心用户流程

```
P2 创建或进入项目
    ▼
P3 配置（左：区域→项目→子要素树；右：标准 + 数据源 + 状态）
    ▼
P3 启动确认（锁定范围与数据源）
    ▼
P3 AI审核中（资料解析 → 标准匹配 → 问题识别与描述及评分）
    ▼
P4 人工复审（修改问题描述 / 打分）     ← 无中转页
    ▼
P5 报告中心（四类产出查看 / 修改 / 独立下载）
```

移动端支线：对「人工录入」类子要素，在 M2 拍照上传后汇入复审可见证据。

# 3. 页面交互关系

```

  │
```

# 4. 核心页面交互说明

## 4.1 P2 审核项目列表

### 页面目标

任务入口：创建、看统计、按状态进入。

### 用户动作

1. 阅读标题下 **一句功能说明**（如：管理质量审核任务并跟进进展）  
2. 查看统计卡片与图表（Mock）  
3. 点击 **创建审核项目** → 系统自动生成项目名称等标识 → 进入 P3  
4. 点击列表项，按状态进入对应页面  

### 系统反馈

| 状态 | 操作文案 | 去向 |
|------|----------|------|
| 配置中 | 继续配置 | P3 配置 |
| AI审核中 | 查看详情 | P3 执行展示 |
| 人工复审中 | 人工确认 | P4 |
| 报告已生成 | 报告中心 | P5 |

### 统计区要求

- 卡片：审核项目总数、AI审核中、人工复审中、已完成审核  
- 图表：审核状态分布、工厂审核情况、问题趋势  

---

## 4.2 P3 审核项目工作台

### 页面目标

完成子要素配置并启动 AI；展示 AI 三步过程；结束后直达复审。

### 4.2.1 配置交互

**无独立「基础信息」步骤。** 创建进入后直接配置范围与数据源。

| 区域 | 交互 |
|------|------|
| 左树 | 展开/选择 **区域 → 项目 → 子要素**；节点显示配置状态（已完成配置 / 缺少数据源 / 未选择） |
| 右配 | 选中子要素后展示：知识库审核标准名称与版本；数据源多选/单选按业务（文件 / 系统 / 人工录入）；系统数据展示字段级说明；文件区为通用上传形态 |
| 完备性 | 启动前检查：至少有需审核子要素且数据源已配齐（规则 Demo 可简化但需有校验提示） |

**人工录入：**

- 在右侧为子要素勾选「人工录入」表示该要素依赖线下表单。  
- 实际拍照上传在移动端完成；Web 侧体现状态「待现场取证 / 已取证」即可（Demo）。  

### 4.2.2 启动确认

1. 用户点击启动  
2. 展示范围与数据源摘要  
3. 弹窗：「启动后审核范围和数据源不可修改」→ 确认后进入 AI审核中并锁定  

### 4.2.3 AI审核中

**步骤（仅此三步在本页）：**

1. 资料解析  
2. 标准匹配  
3. 问题识别与描述及评分  

| 要求 | 说明 |
|------|------|
| 阶段示意 | **静态展示**即可（配置 / AI审核 / 人工复审关系常显），不依赖动态状态变迁动效讲故事 |
| 过程详情 | 每一步展示可讲解内容：解析摘要、命中的具体标准、问题描述与评分依据线索 |
| 结束 | **直接跳转 P4**，不停留「分析完成」中转页 |

---

## 4.3 P4 人工复审工作台

### 页面目标

对 AI 给出的 **问题描述** 与 **打分结果** 进行人工审阅与修改。

### 用户动作

1. 浏览问题列表  
2. 查看依据/标准线索  
3. 修改问题描述、调整打分  
4. 确认后进入报告中心  

### 系统反馈

- 区分 AI 建议与已人工修改  
- 保存后状态保持「人工复审中」，直至进入报告并生成交付物后变为「报告已生成」（Demo 可在首次进入报告中心生成时切换）  

---

## 4.4 P5 报告中心

### 页面目标

承载四类 AI 产出的查看、修改与独立导出。

| 产出 | 交互要点 |
|------|----------|
| ① 问题描述 / BIP 问题管理表 | 查看、人工改、独立下载 |
| ② 问题打分结果 | 查看、人工改、独立下载 |
| ③ 共性问题总结（区域/工厂/集团） | 手动触发生成（Demo）、查看、改、下载 |
| ④ 综合性报告 | 手动触发生成（Demo）、查看、改、下载 |

四者在报告中心为 **并列层级**，均可单独导出；模板由业务方后续提供。

## 4.5 移动端 M1 / M2

| 页面 | 动作 | 反馈 |
|------|------|------|
| M1 | 进入任务 | 任务列表 |
| M2 | 对人工录入类子要素拍照/上传 | 成功后 Web 复审可见来源标记 |

# 5. Demo 演示流程（修订）

### 推荐路径（约 8–12 分钟）

1. **P2**：看统计 → **创建项目**（名称自动生成）  
2. **P3**：左树选子要素 → 看标准 → 配数据源（含说明人工录入+移动端）→ 启动并确认锁定  
3. **P3**：讲解 AI 三步过程详情 → 自动进入 **P4**  
4. **P4**：改一条问题描述或打分  
5. **P5**：切换四类产出，演示独立下载语义  

### 裁剪版

P2 创建 → P3 快速配置并启动 → 看 AI 三步 → P4 改一处 → P5 打开综合报告
