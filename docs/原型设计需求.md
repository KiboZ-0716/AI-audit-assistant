# 1. 原型设计目标

展示完整闭环：

**创建 → 子要素级配置 → AI 三步审核 → 人工复审（改描述/打分）→ 报告中心四类独立导出**

不展示独立驾驶舱；管理与执行同界面。

---

# 2. 核心展示页面

| 优先级 | 页面 | 必做 |
|--------|------|------|
| P0 | P2 审核项目列表 | 是（创建 + 短说明 + 统计卡片/图表 + 四态列表） |
| P0 | P3 审核项目工作台 | 是（左树右配 + 启动锁定 + AI 三步详情） |
| P0 | P4 人工复审工作台 | 是（改问题描述与打分） |
| P0 | P5 报告中心 | 是（四类产出并列、可改、可独立下载语义） |
| P1 | M2 现场取证 | 增强（人工录入类拍照） |

---

# 3. 页面设计输入（摘要）

### P2

- 一句功能说明文案  
- 必须有创建  
- 卡片：总数 / AI审核中 / 人工复审中 / 已完成  
- 图表：状态分布、工厂情况、问题趋势  
- 四态操作：继续配置 / 查看详情 / 人工确认 / 报告中心  

### P3

- 无独立基础信息步；名称自动生成  
- 左树：区域→项目→子要素  
- 右配：知识库标准 + 文件/系统/人工录入 + 状态  
- 数据源与子要素一对一；系统数据展示字段含义；通用上传形态  
- 启动锁定弹窗  
- AI：解析→匹配→识别与描述及评分；展示过程详情；**结束后直达 P4**  
- 生命周期示意静态展示即可  

### P4

- 问题描述与打分的呈现与修改  

### P5

- ① BIP问题管理表 ② 打分结果 ③ 区域/工厂/集团共性总结 ④ 综合报告  
- 均可人工改、独立下载（模板后续提供）  

---

# 4. 原型设计要求

- 遵循 UI 视觉设计规范：专业、可靠、工业化、克制智能  
- 禁止 Admin 模板感、花哨科技风  
- 主路径可点通：P2→P3→P4→P5  

---

# 5. Demo 演示路径

P2 创建 → P3 选子要素配数据源并启动 → 看 AI 三步 → 直达 P4 改一处 → P5 切换四类产出
