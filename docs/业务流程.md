## 1. 流程概述

### 1.1 系统建设背景

在企业制造过程质量审核场景中，审核工作需要依据审核标准，对业务数据及线下文档材料进行分析判断，识别问题并输出审核结论。当前审核对象涉及周期、基地、区域、项目、子要素等多维信息，数据来源包括已数字化的业务系统数据与未数字化的线下文件材料。

### 1.2 业务目标

通过 AI 智能审核助手，实现以下业务目标：

- 依据审核对象自动检索匹配的审核标准
- 支持业务数据自动获取与附件手动上传两种审核数据来源
- 完成文档解析、问题识别、问题分类、严重度打分等智能分析
- 生成结构化审核结果、符合率计算及工厂级问题汇总
- 输出 BIP 问题管理表与审核报告

### 1.3 整体流程作用

整体业务流程以 **确定审核对象** 为起点，经 **数据获取与解析**、**审核标准匹配**、**AI 审核分析**、**指标计算与总结**、**报告生成** 等环节，最终输出结构化审核结果、问题管理表及审核报告，形成完整的审核业务闭环。

## 2. 整体业务流程

整体流程按照以下顺序执行：

```
输入 → 处理 → 分析 → 输出
```

### 2.1 输入阶段

| 维度 | 说明 |
|------|------|
| **用户行为** | 手动选择周期、基地、区域、项目、子要素，确定审核对象 |
| **系统行为** | 接收审核对象信息，作为后续流程的上下文输入 |
| **输出结果** | 已确定的审核对象（周期 + 基地 + 区域 + 项目 + 子要素） |

### 2.2 处理阶段

| 维度 | 说明 |
|------|------|
| **用户行为** | 选择数据来源方式：系统自动审核（业务数据 API）或手动上传附件审核 |
| **系统行为** | 根据选择执行数据获取：通过 HTTP Request 调用业务数据 API（QMS、IQS 等），或通过 File Upload 接收 word/excel/图片附件 |
| **AI 处理逻辑** | 无（本阶段为数据获取） |
| **输出结果** | 原始业务数据或上传文件 |

**数据来源分支说明：**

| 分支 | 说明 |
|------|------|
| 系统自动审核（业务数据 API） | 从 QMS、IQS 等业务系统自动获取数据；区分「已完成数字化」与「未完成数字化」两种状态 |
| 手动上传附件审核 | 上传 word、excel、图片等文件；图片主要指现场拍摄的线下文件照片 |

### 2.3 分析阶段

| 维度 | 说明 |
|------|------|
| **用户行为** | 等待系统完成分析，必要时手动触发符合率计算 |
| **系统行为** | 根据审核对象检索审核标准知识库；执行数据解析（Document Extract）；进行数据处理与审核结果结构化 |
| **AI 处理逻辑** | 问题识别 → 归纳 → 生成描述；问题属性分类；严重度打分；主要问题归纳总结（工厂级）；问题聚类分析 |
| **输出结果** | 结构化审核结果、问题列表、问题分类、严重度评分、工厂级问题汇总 |

### 2.4 输出阶段

| 维度 | 说明 |
|------|------|
| **用户行为** | 查阅审核结果，确认问题清单，获取审核报告 |
| **系统行为** | 执行符合率计算（Code 节点）；生成 BIP 问题管理表；生成审核报告（LLM + 模板） |
| **AI 处理逻辑** | 问题聚类分析（LLM 节点）；报告内容生成（LLM + 模板） |
| **输出结果** | 审核对象列表（已评价/未评价）、符合率、工厂级问题汇总总结、BIP 问题管理表、审核报告 |

## 5. 审核标准匹配流程

### 5.1 审核标准知识库作用

流程图中设有 **审核标准知识库**，用于存储审核标准规则。系统根据已确定的审核对象，从知识库中检索匹配的审核标准，为后续审核判断提供依据。

### 5.2 知识检索方式

| 项目 | 说明 |
|------|------|
| **触发条件** | 审核对象确定后 |
| **检索逻辑** | 根据对象检索审核标准 |
| **对应节点** | Knowledge Retrieval |

### 5.3 标准匹配逻辑

- 输入：已确定的审核对象（周期、基地、区域、项目、子要素）
- 处理：根据审核对象从审核标准知识库中检索匹配的标准
- 输出：匹配到的审核标准，供审核判断环节使用

### 5.4 审核结果生成方式

匹配到审核标准后，结合已结构化的审核数据，进入 AI 审核判断环节，生成结构化审核结果。

## 7. 指标计算与总结流程

### 7.1 指标计算方式

流程图中的核心指标为 **符合率**，需在子要素级别先完成打分后计算。

### 7.2 计算条件

- 需先完成子要素级别的打分
- 依据当前已有的审核子要素进行符合率计算
- 每次完成新的审核项后更新
- **业务答复：手动触发**

### 7.3 计算规则

**公式 1：区域符合率**

```
区域符合率 = （各条款评分总和 / 条款评价项 × 10）× 100%
```

**公式 2：工厂总体符合率**

```
工厂总体符合率 = （∑各区域条款评分总和 / ∑各区域条款评价项 × 10）× 100%
```

### 7.4 特殊处理逻辑

- 若某区域不涉及或未评价的条款，则标记为 **NA**，不计入符合率计算
- 符合率计算通过 Workflow App 执行，每次完成新的审核项后可手动触发更新

## 8. AI 问题总结规则

### 8.1 输入

```
问题列表（problem_list）
```

每个问题包含问题描述信息。

### 8.2 AI 处理

对应 **问题聚类分析** LLM 节点，处理规则如下：

| 规则编号 | 规则内容 |
|----------|----------|
| 1 | 不逐条罗列问题 |
| 2 | 将逻辑相近的问题合并 |
| 3 | 输出 3-5 个主要问题 |
| 4 | 每个问题按照「问题名称 + 具体表现」的格式输出 |
| 5 | 使用审核报告语言 |
| 6 | 归纳方向参考专家经验输入（流程图标注：最好有一些专家经验输入） |

**归纳方向参考（流程图列出的分类方向）：**

| 分类方向 | 参考维度 |
|----------|----------|
| 问题管理类 | 问题闭环不足、整改跟踪不足 |
| 流程类 | 流程机制不完善、执行不到位 |
| 数据类 | 数据完整性不足、数据应用不足 |
| 专项管理类 | 专项治理不足 |

**业务答复：** 可以给历史数据自行训练总结。

### 8.3 输出

**主要问题总结**（工厂级），用于工厂级问题汇总总结环节。

## 9. 审核报告生成流程

### 9.1 报告生成输入

- 结构化审核结果
- 问题列表及问题描述
- 问题分类与严重度评分
- 工厂级主要问题总结
- 符合率计算结果
- BIP 问题管理表数据

### 9.2 报告内容

报告内容采用结构化四段式组织，涵盖本次审核的分析结论：

- **审核概述**：审核对象信息、范围/周期等基本信息与符合率指标
- **亮点工作**：本次审核中形成的优秀实践/亮点归纳
- **主要问题**：工厂级主要问题总结（3-5 个主要问题），包含问题分类与严重度表现
- **下一步要求**：针对主要问题提出的整改要求与建议，并关联 BIP 问题管理信息

### 9.3 生成方式

| 项目 | 说明 |
|------|------|
| **对应节点** | LLM + 模板 |
| **处理方式** | 基于 LLM 生成报告内容，结合模板输出结构化报告 |

### 9.4 最终输出

- 审核报告文档
- 关联输出：BIP 问题管理表、审核对象列表（已评价/未评价）

## 10. Workflow 节点说明

流程图中的业务节点与 Dify Workflow 技术节点对应关系如下：

| 节点 | 作用 | 输入 | 输出 |
|------|------|------|------|
| Workflow Trigger（Workflow 输入） | 审核任务创建，接收审核对象及任务参数 | 用户选择的周期、基地、区域、项目、子要素 | 审核任务上下文 |
| HTTP Request | 数据获取，从业务系统自动拉取审核数据 | 审核对象、业务系统 API 参数 | QMS、IQS 等业务数据 |
| File Upload | 文件上传，接收手动上传的审核附件 | word、excel、图片文件 | 上传文件 |
| Document Extract | 数据解析，对文档及图片进行内容提取 | 上传文件或原始文档 | 解析后的文本/结构化内容 |
| Knowledge Retrieval | 规则匹配，根据审核对象检索审核标准 | 审核对象 | 匹配的审核标准 |
| LLM 节点（审核判断） | 审核判断，识别审核问题 | 结构化数据 + 审核标准 | 问题识别结果 |
| LLM 节点（问题评分） | 问题评分，对问题进行严重度打分 | 问题列表 | 严重度评分 |
| LLM 节点（问题生成） | 问题生成，归纳并生成问题描述 | 问题识别结果 | 结构化问题描述 |
| LLM 节点（问题总结） | 问题总结，工厂级问题聚类归纳 | 问题描述列表（problem_list） | 3-5 个主要问题总结 |
| Code 节点 | 符合率计算，按公式计算区域及工厂符合率 | 子要素级别评分数据 | 区域符合率、工厂总体符合率 |
| LLM + 模板 | 报告生成，生成审核报告 | 审核结果、问题总结、符合率等 | 审核报告 |

## 11. 系统边界

### 11.1 一期实现范围

依据流程图，一期实现范围包括：

| 范围 | 内容 |
|------|------|
| 审核对象确定 | 手动选择周期、基地、区域、项目、子要素 |
| 数据获取 | 业务数据 API 自动获取（QMS、IQS 等）+ 附件手动上传（word/excel/图片） |
| 数据解析 | 文档解析、图片识别、数据结构化处理 |
| 标准匹配 | 审核标准知识库检索 |
| AI 审核分析 | 问题识别、问题描述生成、问题分类、严重度打分 |
| 问题总结 | 工厂级问题聚类归纳（3-5 个主要问题） |
| 指标计算 | 区域符合率、工厂总体符合率（手动触发） |
| 结果输出 | 审核对象列表、BIP 问题管理表、审核报告 |
| 移动端 | 兼容移动端；支持上传图片；支持针对问题清单在移动端修改（文本/语音） |

### 11.2 人工参与环节

| 环节 | 人工参与方式 |
|------|-------------|
| 审核对象确定 | 用户手动选择周期、基地、区域、项目、子要素 |

| 审核结论确认 | 专业人员对 AI 识别结果进行确认 |

### 11.3 暂不实现内容

依据流程图中的业务答复标注，以下内容暂不实现或不体现：

| 项目 | 业务答复 |
|------|----------|
| 审核类型区分 | 不需要区分 BIP 审核/专项审核，不需要体现 |
| 评分规则细化 | 业务无法细化评分规则，可提供案例，技术侧考虑微调方案 |
| 专家经验输入 | 问题总结归纳方向可参考专家经验，业务答复可通过历史数据自行训练总结 |
