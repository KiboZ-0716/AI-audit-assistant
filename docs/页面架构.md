## 1. 页面架构说明

### 1.1 项目定位

当前交付物是 **客户演示 Demo**，不是生产系统。页面服务于客户展示、方案演示与业务流程验证。

### 1.2 设计目标

| 目标 | 说明 |
|------|------|
| 业务完整可演 | 创建 → 配置 → AI 审核 → 人工复审 → 报告中心 |
| 理解成本低 | 一眼可识别为质量审核 AI 工作平台 |
| 结构克制 | 取消与核心流程无关的独立驾驶舱；能力合并到任务主链 |
| 人机协同清晰 | AI 建议可改，正式结论经人工复审 |

### 1.3 角色与页面（v3.0）

| 角色 | 页面可见内容 | 差异点 |
|------|--------------|--------|
| 审核管理人员 | 与实际审核人员 **同一套页面与流程** | 仅 **数据权限范围** 不同（可见项目/工厂范围更大） |
| 实际审核人员（Web） | 列表 → 工作台 → 人工复审 → 报告中心 | 执行主用户 |
| 实际审核人员（移动端） | 现场任务与拍照上传 | 服务「人工录入/线下表单」类子要素的现场取证 |

> **已取消独立「审核驾驶舱」模块。** 原管理态势类信息（状态分布、工厂情况、问题趋势等）收敛到 **审核项目列表页** 的统计区展示。

### 1.4 模块 ≠ 页面

| 层级 | 含义 |
|------|------|
| 一级模块 | 导航能力域 |
| 独立页面 | 路由级主界面 |
| 页内能力 | 步骤、树、分区、弹层、上传区等 |

## 2. 系统页面结构

### 2.1 独立页面结构树

```
AI 智能审核助手 Demo
│
├── 【Web 端】
│   ├── 1. 审核项目管理
│   │   ├── P2 审核项目列表          ← 任务入口 + 统计 + 创建
│   │   └── P3 审核项目工作台        ← 配置 / AI 审核中（无「分析完成」中转页）
│   │
│   ├── 2. 人工复审
│   │   └── P4 人工复审工作台        ← 问题描述与打分结果的审阅/修改
│   │
│   └── 3. 报告中心
│       └── P5 报告工作台            ← 四类 AI 产出独立查看/修改/导出
│
└── 【移动端】
    └── 4. 移动端现场取证
        ├── M1 我的审核任务
        └── M2 现场工作台            ← 线下表单拍照上传等（对应人工录入类数据源）
```

> Web **4 个独立页面**（P2–P5）+ 移动端 **2 个**。原 P1 驾驶舱删除。

### 2.2 项目状态（全局四态）

| 状态 | 说明 | 列表主操作 |
|------|------|------------|
| 配置中 | 范围与子要素数据源尚未启动 | 继续配置 |
| AI审核中 | AI 正在执行解析/匹配/识别与评分 | 查看详情 |
| 人工复审中 | AI 已产出，待人工修改确认 | 进入人工确认 |
| 报告已生成 | 报告中心产出可用 | 进入报告中心 |

### 2.3 端到端路径

```
P2 列表（创建 / 统计 / 按状态进入）
    │
    ▼
P3 工作台
    │  配置：左树（区域→项目→子要素）+ 右配（标准 / 数据源 / 状态）
    │  启动确认（锁定范围与数据源）
    │  AI审核中：资料解析 → 标准匹配 → 问题识别与描述及评分
    │  （结束后直接进入人工复审，无中转页）
    ▼
P4 人工复审工作台
    │  审阅/修改问题描述与打分结果
    ▼
P5 报告中心
    │  四类产出：BIP问题管理表 / 打分结果 / 共性问题总结 / 综合性报告
    │  均可独立下载；均支持人工复审修改
```

## 3. 页面模块说明

### 3.1 P2 审核项目列表

| 项 | 说明 |
|----|------|
| **页面目的** | 审核任务总入口；承接创建、进展查看与按状态进入后续工作台 |
| **标题说明** | 菜单「审核项目」下保留 **一句短说明**，说明本页用于管理与进入审核任务（文案示例：管理质量审核任务并跟进进展） |
| **必须能力** | **创建审核项目**入口（自动生成项目名称等基础标识，无独立「基础信息」配置步） |
| **统计区** | **数据卡片**：审核项目总数、AI审核中、人工复审中、已完成审核（对应「报告已生成」） |
| **图表区** | 审核状态分布、工厂审核情况、问题趋势等高价值统计（Demo 用 Mock） |
| **列表** | 展示项目关键信息、四态状态、进度说明、按状态进入 |

创建后：生成项目并进入 P3 配置（名称自动生成，不单独手填「基础信息页」）。

---

### 3.2 P3 审核项目工作台

**页面职责：** 完成子要素级配置并启动 AI；展示 AI 审核过程；结束后直达人工复审。

#### 3.2.1 配置组织方式（无独立基础信息步）

创建时自动带出/生成必要标识（如项目名称），用户在本页直接进入 **审核范围与数据源配置**。

**核心布局：左树 + 右配**

| 区域 | 内容 |
|------|------|
| **左侧** | 审核对象树：**区域 → 项目 → 子要素**（示例：零部件 → 质量数据采集 → 质量监视测量设备配置） |
| **右侧** | 当前选中 **子要素** 的配置：① 匹配到的知识库审核标准（如「设备管理审核规则 V1.0」）；② 数据源选择（文件资料 / 系统数据 / 人工录入）；③ 配置状态 |
| **左侧状态** | 子要素节点展示：已完成配置 / 缺少数据源 / 未选择 等 |

#### 3.2.2 数据源与子要素一对一

- 数据源挂在 **子要素** 上，禁止项目级「一批文件打天下」的批量匹配假设。  
- 系统数据需体现具体取数含义（如问题记录时间等字段级说明，Demo 可用 Mock 字段名）。  
- 文件资料采用通用上传交互形态（非纯文字入口）。  

#### 3.2.3 「人工录入」与移动端关系

| 概念 | 说明 |
|------|------|
| 人工录入（数据源类型） | 该子要素无法依赖上传文件或系统自动取数，依赖线下表单 |
| 移动端 | 线下表单的现场取证方式（如拍照上传），作为人工录入类子要素的数据采集入口 |
| Web 配置 | 为子要素勾选「人工录入」并看到标准与状态；具体影像采集在移动端完成并汇入后续复审 |

#### 3.2.4 启动确认

- 展示已选范围与各子要素数据源完备情况。  
- 启动后弹窗确认：**审核范围与数据源不可再修改**。  

#### 3.2.5 AI 审核中（执行展示）

执行链路（本页完成）：

```
资料解析 → 标准匹配 → 问题识别与描述及评分
```

| 要求 | 说明 |
|------|------|
| 生命周期示意 | **静态展示**「配置 / AI审核 / 人工复审」等阶段关系即可，不做炫技式状态变迁动效 |
| 过程详情 | 每一步宜展示可讲解的过程信息（如解析结果摘要、匹配到的具体标准、问题描述与评分的依据线索） |
| 结束后 | **不设「分析完成」中转页**，直接进入 P4 人工复审 |

> 工厂级/集团级共性问题总结、综合性报告等 **不在本页完成**，见报告中心。

---

### 3.3 P4 人工复审工作台

| 项 | 说明 |
|----|------|
| **页面目的** | 对人机协同产出的 **问题描述** 与 **打分结果** 进行审阅与修改 |
| **核心任务** | 查看 AI 建议 → 修改问题描述 / 打分 → 确认后进入报告中心 |
| **不承担** | 四类正式报告排版与独立导出（在 P5） |

---

### 3.4 P5 报告中心

整体方案中 AI 介入的 **四类可独立产出**（均可单独导出下载；均支持人工复审修改；模板后续提供）：

| 序号 | 产出 | 说明 |
|------|------|------|
| ① | 问题描述 / BIP 问题管理表 | 基于识别结果形成问题清单类产出 |
| ② | 问题打分结果 | 针对问题的评分结果呈现与导出 |
| ③ | 共性问题总结 | 按区域级 / 工厂级 / 集团级汇总共性问题 |
| ④ | 综合性报告 | 汇总性审核报告 |

| 项 | 说明 |
|----|------|
| **页面目的** | 四类产出的查看、人工修改、独立下载 |
| **触发** | 可在报告中心按类型手动触发生成/刷新（Demo 用 Mock） |
| **与 P4 关系** | P4 侧重问题描述与打分的复审修改；P5 承载四类正式交付物 |

---

### 3.5 移动端（M1 / M2）

| 页面 | 职责 |
|------|------|
| M1 | 我的审核任务 |
| M2 | 现场工作台：针对「人工录入」类子要素进行拍照等线下取证上传 |

不在移动端做完整配置、复审定稿与报告导出。

## 4. 页面与业务流程关系

| 业务环节 | 页面 |
|----------|------|
| 创建任务 | P2 |
| 范围与子要素数据源配置、启动 | P3 |
| AI 解析 / 匹配 / 识别描述及评分 | P3 |
| 人工修改问题描述与打分 | P4 |
| 四类报告生成、修改、导出 | P5 |
| 线下表单拍照取证 | M2 → 汇入复审链路 |

## 5. Web 与移动端关系

| 端 | 定位 |
|----|------|
| Web | 主演示舞台：配置、AI 审核、复审、报告 |
| 移动端 | 人工录入类子要素的现场取证补充 |

| 变更 | 说明 |
|------|------|
| 删除 P1 驾驶舱 | 统计并入 P2 |
