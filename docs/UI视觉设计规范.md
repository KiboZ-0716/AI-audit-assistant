# 1. 产品视觉定位

## 1.1 产品应给用户的感觉

本产品面向汽车制造场景下的质量审核专业人员与管理人员。视觉气质应传达：

| 感受 | 含义 |
|------|------|
| **专业** | 像资深质量工程师的工作台，而不是通用办公工具 |
| **可靠** | 信息稳定、结论可追溯，适合严肃审核决策场景 |
| **工业化** | 贴近制造与质量管理语境，克制、秩序、可执行 |
| **智能化** | 能感知 AI 在辅助分析，但智能感服务于业务，不喧宾夺主 |
| **高效** | 一眼能抓住任务进展、问题重点与下一步动作 |

一句话定位：

> **冷静、可信的企业级 AI 审核工作平台** —— 有智能辅助感，但首先是专业质量工具。

## 1.2 目标用户的视觉预期

| 角色 | 打开产品时期望看到的感觉 |
|------|--------------------------|
| 实际审核人员 | 清晰的工作推进感、分析结果可读、确认动作明确 |
| 审核管理人员 | 全局可控、风险可见、数据可信，不花哨 |
| 客户（Demo 观众） | 立刻理解「这是质量审核 AI 方案」，而非后台管理系统模板 |

## 1.3 与「普通后台」的差异感（视觉层面）

- 更强调 **任务推进与分析结论**，弱化「表单仓库」印象  
- 更强调 **AI 建议与人工确认的关系**，弱化纯 CRUD 操作感  
- 更强调 **工业质量语境的克制美学**，弱化互联网运营后台的营销感

# 2. 行业视觉参考方向

## 2.1 场景与行业特征

| 维度 | 特征 | 对视觉的启示 |
|------|------|--------------|
| 汽车制造 | 精密度高、流程严谨、现场与体系并重 | 秩序感、结构感、少装饰噪音 |
| 质量管理 | 标准、符合率、问题闭环、证据链 | 状态语义清晰、结论层级分明 |
| 工业软件 | 长时间使用、信息连续处理 | 耐看、低干扰、稳定可读 |
| 企业级系统 | 多角色、重效率、重信任 | 信息架构清楚，避免娱乐化表达 |

## 2.2 适合的视觉风格方向

推荐靠近以下气质组合：

1. **现代工业软件**  
   干净、结构化、以内容与状态为主，装饰让位于信息。

2. **企业专业分析工具**  
   类似高质量分析工作台：强调层级、对比与结论可读，而非卡片橱窗。

3. **克制的智能产品**  
   AI 存在感应通过「分析过程、建议标识、依据呈现」体现，而不是霓虹、光效、科幻皮肤。

## 2.3 可参考的气质来源（方向性，非抄袭对象）

- 高端制造 / 质量体系类工作软件的冷静界面  
- 专业审计、合规、检测类工具的证据与结论表达方式  
- 现代企业生产力工具的清晰排版与稳定导航  

不建议作为主参考：

- 消费级 AI 聊天产品的活泼/拟人化风格  
- 通用 Admin / 中台模板皮肤  
- 游戏化、赛博朋克、强未来感仪表盘

# 3. 整体设计关键词

## 3.1 色彩倾向

定义方向，不锁定具体色值：

| 关键词 | 方向说明 |
|--------|----------|
| 主色气质 | 偏冷静、可信的企业色系；建议以深蓝/青灰蓝/工业蓝一类「专业信任色」为品牌主倾向 |
| 中性色 | 以灰阶承载大部分界面结构与文本，保证长时间阅读舒适 |
| 语义色 | 成功、警告、风险、进行中等状态用有限、明确的语义色，服务审核判断，不做装饰性彩虹 |
| 背景气质 | 浅色、干净、稳定为主（Demo 默认日间专业场景）；避免为了「科技感」默认深色皮肤 |
| 色彩克制 | 同屏主强调色宜少；AI 相关强调应可识别，但不超过业务主操作层级 |

不鼓励的色彩倾向：

- 紫粉互联网感、霓虹赛博感  
- 大面积高饱和撞色  
- 为装饰而做的复杂多色渐变背景  

## 3.2 氛围

| 关键词 | 说明 |
|--------|------|
| 冷静 | 少情绪化插画与口号式视觉 |
| 清晰 | 层级明确，重点可被讲解时一眼指出 |
| 可信 | 边框、分割、对齐体现秩序，避免漂浮无根的装饰块 |
| 进取但不喧闹 | 可以有现代感，但节奏平稳，适合企业汇报演示 |

## 3.3 信息密度

| 场景 | 密度建议 |
|------|----------|
| 驾驶舱 | 中密度：总览可读，不堆砌无意义指标墙 |
| 项目配置 / 执行 | 中高密度：步骤与状态清楚，留白服务阅读而非空洞 |
| 结果确认 | 高密度但有层级：问题列表为主，依据与建议可展开查看 |
| 报告 | 中密度：接近正式文档阅读体验，强调结构章节感 |

原则：**密度服务专业效率，不为「高级感」刻意留白到空；也不为「功能多」挤成报表丛林。**

## 3.4 品牌感

品牌感应来自：

- 稳定的专业气质与一致的状态语言  
- AI 辅助在关键节点被看见（执行中、建议、依据）  
- 审核业务语义（符合率、问题、确认、报告）被视觉正确强调  

品牌感不应来自：

- Logo 墙、 Slogan 横幅堆叠  
- 过度吉祥物/插画叙事  
- 与汽车质量无关的潮流皮肤  

Demo 场景下，品牌感 = **「一看就是质量审核 AI 工作平台」**。

## 3.5 造型与质感关键词（原则级）

- 造型：克制圆角、清晰边界、对齐严谨  
- 质感：平整、干净，轻微分层即可；避免多层阴影营造「悬浮卡片展览」  
- 动效：服务状态理解（进度、切换、确认反馈），短、稳、可预期

# 4. AI 产品特征表达

## 4.1 要体现的智能特征

| 特征 | 视觉表达方向 |
|------|--------------|
| AI 能力存在 | 在分析、建议、生成处有稳定可识别的「AI 产出」标识与语气区分 |
| 自动分析 | 执行过程可见：阶段、进度、正在处理的含义可被讲解 |
| 智能辅助 | AI 内容呈现为「建议 / 初稿 / 依据」，视觉权重低于或平行于「人工确认」 |
| 人机协同 | 明确区分：AI 建议态、待确认态、已确认/已修订态；让客户看见人的决策痕迹 |

## 4.2 推荐的表达策略

1. **过程可见**  
   AI 执行不是黑盒加载，而是可理解的阶段推进。

2. **建议可辨**  
   AI 给出的问题、评分、报告初稿，应能与人工确认后的结论区分。

3. **依据可追**  
   展示「为什么这么判断」的入口与呈现，强化专业可信，而非只给结论标签。

4. **确认优先**  
   最终生效相关的确认动作，在视觉层级上应稳定、明确，不弱于 AI 炫技区域。

## 4.3 避免的 AI 视觉误区

| 避免 | 原因 |
|------|------|
| 过度科技化 | 光效、粒子、科幻仪表会削弱工业质量可信度 |
| 娱乐化 | 表情化、俏皮插画、游戏徽章会降低严肃审核语境 |
| AI 装饰喧宾夺主 | AI 标签满屏会让产品像「AI 概念页」而非工作台 |
| 假装全自动无人值守 | 与「人工最终确认」产品原则冲突 |
| 神秘黑盒 | 只有结果没有过程与依据，客户难以建立信任 |

## 4.4 Demo 讲解友好性（视觉）

客户演示时，视觉应让讲解者能自然指向：

- 「这里是 AI 正在分析」  
- 「这里是 AI 建议」  
- 「这里是人工确认后生效」  

三者关系清楚，比任何装饰性科技元素更重要。

# 5. B 端设计原则

## 5.1 易读

- 文本层级清楚：标题、结论、说明、辅助信息可分辨  
- 关键数字与状态具备足够对比度  
- 长列表/问题清单以扫描阅读为目标，避免装饰打断阅读节奏  

## 5.2 稳定

- 同类型信息在不同页面保持一致的表达方式  
- 状态颜色与命名稳定，不随页面情绪化变化  
- 布局节奏可预期，减少「每个页面一种性格」  

## 5.3 高效

- 主任务路径上的关键动作易于发现  
- 减少为展示而展示的视觉模块  
- 让用户（与讲解者）快速回到「下一步该做什么」  

## 5.4 数据清晰

- 指标、符合率、问题数量、进度等应可被快速比较与口述  
- 图表与统计服务判断，不做无意义美化图表墙  
- 风险与异常应用语义方式被看见，而不是淹没在平均数据中  

## 5.5 企业软件习惯对齐

- 尊重专业用户对「可控、可追溯、可确认」的偏好  
- 交互反馈明确：成功、失败、进行中、需确认  
- 移动端现场场景保持同样专业克制，只是更适合单手高效完成采集与补充

# 6. 禁止设计方向

以下方向明确避免，作为 Demo 与高保真设计的负面清单：

## 6.1 形态禁止

- **普通 Admin 模板感**（左侧厚重菜单 + 右侧表格堆砌的默认后台长相）  
- **通用 SaaS 后台模板感**（同质化卡片仪表盘、无业务识别度）  
- **为组件而组件** 的展示型页面（看起来像 UI Kit 预览，不像审核工作台）  

## 6.2 视觉禁止

- 花哨渐变背景、霓虹光效、玻璃拟态炫技  
- 过度卡片堆叠、多层阴影、到处圆角胶囊造成的「运营活动页」感  
- 高饱和多色块无语义拼贴  
- 装饰性插画主导首屏，挤压业务信息  

## 6.3 动效禁止

- 炫技动画、长时长转场、无意义悬浮/呼吸光  
- 干扰阅读的持续动效  
- 用动画掩盖信息结构不清  

## 6.4 表达禁止

- 娱乐化、消费互联网活泼语调视觉  
- 过度拟人 AI 形象成为视觉主角  
- 深色赛博皮肤作为默认企业演示主风格（除非客户明确要求暗色方案）  

## 6.5 设计自检问题

完成任一高保真页面后，可用以下问题自检：

1. 去掉 Logo 后，是否仍像「质量审核 AI 工作平台」？  
2. 客户能否在 3 秒内指出：进展、AI 建议、需确认项？  
3. 是否出现了可删掉且不影响理解的装饰？  
4. 是否看起来像随便换皮的 Admin 模板？  

若 1 否、2 否、3 是、4 是 —— 需要回到本规范重做方向校准。

# 7. 使用说明（给 Figma / 前端）

## 7.1 本规范的用途

- 统一视觉方向与审美边界  
- 减少 Demo 各页面风格漂移  
- 作为评审「像不像本产品」的标准  

## 7.2 本规范不做什么

- 不规定具体色值表、字体文件名单、间距刻度  
- 不规定页面布局与组件结构  
- 不输出 CSS/代码  
- 不替代后续单页高保真设计决策  

设计师与前端在遵循本方向的前提下，可进行具体创作；若需沉淀色板与字体尺度，应在下一阶段「设计系统/高保真」中单独产出。
